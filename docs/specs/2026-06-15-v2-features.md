# Spec: Doneline v2 feature batch

_Date: 2026-06-15_

## Items
1. **Multi-day / ranged events** — events can span a start→end date. Add-event UI
   gets an optional end date; calendar renders the event across each covered day.
2. **Day detail popup** — clicking a calendar day opens a panel listing that day's
   events + todos (with owner), an **Add event** button, and **delete** per item.
3. **Recurring todos & events** — repeat rule: `daily` or specific `weekdays`.
   A fresh instance is generated for each matching day (checked off independently).
   Set in the add-todo / add-event modals and when adding a todo to a goal.
4. **Todo lifecycle**
   - Done todos stay (checked) until **midnight**, then become `archived` (hidden
     from lists, kept in DB).
   - Overdue, still-open todos render **red "Past due"** and remain.
   - **Settings → Archive**: lists archived done todos; items older than **14 days**
     are permanently purged automatically.
5. **Sounds** (Web Audio, generated — no files): focus phase chime, last-5-second
   countdown ticks, todo check-off ding, event-reminder sound, subtle UI clicks.
   Global mute toggle in Settings.
6. **Focus transition animation** — on focus⇄break switch, the phase label eases in
   at center then rises above the timer; chime on switch; ticks for last 5 seconds.
7. **Music** — channels grouped into Lofi, Piano/Classical, Rain/Nature, Jazz.
   Streamed where reliable; Rain/Nature generated offline. Audio errors already
   handled (timer unaffected).

## Data / schema changes
- `todos`: add `archived INTEGER DEFAULT 0`, `recurrence TEXT` (JSON or null),
  `recur_parent TEXT`. `events`: add `recurrence TEXT`.
- Recurrence JSON: `{ freq: 'daily' | 'weekly', days?: number[] }` (days 0–6, Sun=0).
- Generation: on app start + each day, for each recurrence template create the
  instance for today if missing (dedupe by template id + date).
- Archive sweep: on start + hourly, set `archived=1` on todos completed before
  today; hard-delete archived rows whose `completed_at` is > 14 days old.

## Architecture notes
- New `core/recurrence.ts` (template→instance generation) and archive/purge helpers
  in `todos.ts`. Sound + music live in renderer (`lib/sound.ts` extended,
  `lib/audioFx.ts` for generated SFX). Day popup = new component in CalendarView.
- Keep files small; no backend/MCP contract breakage (additive columns only).

## Out of scope
- Every-N-days recurrence (not requested), monthly recurrence, drag-resize on
  calendar, custom user stream URLs.

## Success criteria
- Repeating items appear each correct day; done todos archive at midnight and show
  in the Archive (purged after 14 days); overdue todos show red; calendar day click
  shows everything with add/delete; multi-day events span days; focus has a smooth
  transition + sounds; music has the four vibes with a working mute.
